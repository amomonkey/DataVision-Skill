# 散点图（Scatter）

## Overview

散点图用于展示**两个连续变量之间的关系**或**多维数据分布**。每个点的位置由 X / Y 坐标决定，大小可映射第三维（数值大小），颜色可映射第四维（类别 / 连续值），最多可同时表达 4 个维度。

适用场景：

- 相关性分析（X-Y 两轴的连续变量关系）
- 多维数据分布（位置 + 大小 + 颜色）
- 聚类 / 分组（按类别上色，观察分布聚类）

---

## 数据结构

每个数据点是一个数组 `[x, y, value?, category?]`：

- `x` / `y`：坐标轴位置
- `value`（可选）：第三维，用于映射点大小
- `category`（可选）：第四维，用于映射颜色 / 分组

也支持对象形式 `{ x, y, value, category, ... }`。

---

## 符号

可选符号形状：

| 符号 | 用途 |
| --- | --- |
| 圆形（默认） | 常规散点 |
| 矩形 | 长方形数据点 |
| 三角形 | 强调指向 |
| 菱形 | 强调对称 |
| 标记点（pin） | 特殊位置标识 |
| 箭头 | 方向 / 流向 |

---

## 大小

支持三种大小映射方式：

| 方式 | 说明 |
| --- | --- |
| 固定大小 | 所有点同一大小（如 20px） |
| 大小范围 | 按数据 `value` 映射到 `[最小值, 最大值]` |
| 动态计算 | 自定义函数计算每个点大小 |

**移动端推荐范围**：最小 **12px** / 最大 **64px**。

---

## 颜色映射

| 映射类型 | 说明 |
| --- | --- |
| 类别映射 | 按 `category` 字段分组着色，每类取一个色板色 |
| 连续映射 | 按数值在两端颜色间渐变（如 `[低色, 高色]`） |

颜色取自 [tokens.md — 可视化色板](../tokens.md)。

---

## 数据标签

- 默认**不显示**（点多时易碰撞）
- 启用后支持模板格式化（如显示 `category` 名）
- 标签自动布局：开启后自动调整位置避免重叠

---

## 交互能力

| 能力 | 说明 |
| --- | --- |
| 临近搜索 | 鼠标悬停 / 点击时搜索附近一定半径内的点；支持限制最大结果数 |
| 多选 | 框选或多次点击选中多个点 |
| 保持高亮 | 选中状态长期保持，不随鼠标移开消失 |
| 自动隐藏 | 静止一定延迟后自动隐藏 Tooltip / 高亮（如 2000ms） |

---

## 实现要点（库无关）

- **大小用范围映射，不要写死**：数据 `value` 差异大时，固定大小会丢失维度信息；映射到一个 `[min, max]` 范围更直观。
- **类别少用类别色，数值多用连续色**：类别 ≤ 6 用顺序色板分配；连续数值用两端颜色渐变。
- **临近搜索半径需要按数据密度调**：数据稀疏时半径大、密集时半径小，默认从 50px 起调。
- **数据量大时降采样或自动隐藏标签**：避免渲染卡顿和标签糊成一团。
- **最小点不小于 12px**：移动端点击热区底线，否则用户难以选中。

---

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 点大小用范围映射（移动端 `[12, 64]` 推荐），不要写死单一大小 |
| ✅ | 颜色从可视化色板取，不要硬编码 |
| ✅ | 启用临近搜索改善密集数据的可点击性 |
| ✅ | 类别色 vs 连续色按数据性质选 |
| ❌ | 不要默认显示所有点的标签——点多必碰撞 |
| ❌ | 不要让点大小 < 12px（移动端无法点击） |
| ❌ | 不要用散点图展示纯类别数据（那是柱图的场景） |
