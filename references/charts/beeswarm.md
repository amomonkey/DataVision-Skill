# 蜂群图（Beeswarm）

## Overview

蜂群图用于展示**一维数据的分布**——所有数据点沿一个轴排列（按值映射），相同值的点向两侧偏移避免重叠，形成"蜂群"形态。

典型应用：

- 单维度分布（如薪资分布、年龄分布）
- 类别对比（多组数据沿不同 Y 位置对比分布）
- 异常值识别（远离主群的点）

与散点图的区别：散点图是二维关系；蜂群图是一维分布 + 重叠规避，**专注于一个变量的分布形态**。

---

## 数据结构

蜂群图基于 ECharts graph series 实现，数据是节点数组：

```
[
  { value: 12, symbolSize: 8, ... },
  { value: 18, symbolSize: 10 },
  { value: 12, symbolSize: 8 },
  ...
]
```

- `value`：数值（决定主轴位置）
- `symbolSize`：点直径（决定半径）

---

## 布局类型

| 类型 | 说明 |
| --- | --- |
| `static`（静态） | 默认。按值映射到轴位置，垂直方向不动，水平方向偏移避免重叠 |
| `force`（力导向） | 用 d3-force 算力学模拟，点之间互斥力 + 朝目标位置的吸引力 |
| `absolute`（绝对） | 完全按数据指定坐标，无自动布局 |

---

## Static 布局算法

paradigm-chart 的 static 布局逻辑（基于代码）：

1. 节点按值映射到 Y 轴位置（沿主轴）
2. 节点按**半径从大到小**排列（大点先放，避免后放的小点被挤出）
3. 每个点初始 X = 容器中心；若与已放置的点重叠，按 ±1, ±2, ±3... 偏移 X，直到不重叠
4. 偏移方向左右交替（先 +1，再 -1，再 +2，再 -2...），形成蜂群左右对称形态

---

## 节点样式

| 元素 | 默认 |
| --- | --- |
| 形状 | 圆形（symbol: circle） |
| 大小（`symbolSize`） | 按数据点指定，常用 6-12px |

### 数据标签策略

| 规则 | 说明 |
| --- | --- |
| **默认不显示** | 点密集时全部显示标签必然重叠，默认隐藏 |
| **hover 显示** | 鼠标悬停某点时仅显示该点标签 |
| **极值点显示** | 或仅在最大 / 最小点等"关键点"显示标签，其余隐藏 |

> 不要默认全显——蜂群图的核心在"分布形态"而非"逐点数值"，标签应让位于点的位置可读性。
| 颜色 | 系列色 / 按类别映射 |

---

## 主轴

| 轴 | 用途 |
| --- | --- |
| Y 轴 | 主轴，承载数据 `value` |
| X 轴 | 偏移方向（不承载数据） |

> 蜂群图可水平或垂直，paradigm-chart 实现里 Y 为主轴。可视场景下也可旋转。

## 实现要点（库无关）

- **大点先放**：布局算法必须按半径降序处理点，先放大的、再放小的，否则小点会被大点挤到边缘失序。
- **偏移方向左右对称**：偏移序列 +1, -1, +2, -2... 而不是单方向 +1, +2, +3...，让蜂群左右对称。
- **节点不能重叠**：判断距离 < r1 + r2 即重叠，逐步外推；这是蜂群图的硬约束。
- **数据量适中**：节点 > 数百时计算和渲染都吃力，考虑降采样或聚类。
- **类别用颜色区分**：多类别数据沿不同 Y 位置 + 不同颜色，能在一张图里对比多组分布。

---

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 节点按半径从大到小排列后再布局 |
| ✅ | 偏移方向左右对称（±1, ±2, ...） |
| ✅ | 节点不重叠（距离 ≥ 半径和） |
| ✅ | 数据量大时降采样或改用密度图 |
| ❌ | 不要让节点重叠（破坏蜂群可读性） |
| ❌ | 不要在节点数 > 500 时硬上 static 布局——性能差 |
| ❌ | 不要用蜂群图展示二维关系（那是散点图） |
