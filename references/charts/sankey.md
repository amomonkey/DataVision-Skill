## Overview

桑基图用于展示**流向 / 流量分配**——节点表示状态或类别，节点之间的边（带状曲线）表示流量，**边宽度按流量大小映射**。

典型应用：

- 资金 / 能源 / 物流流向（A 行业 → B 行业 → C 类产品）
- 用户路径分析（来源页 → 中间页 → 转化页）
- 多阶段分配（总收入 → 各部门 → 各项目）

paradigm-chart 的 sankey 实现为 **SankeyTree**（桑基树形布局），在 ECharts 原生 sankey 基础上加了树状布局能力。

---

## 数据结构

桑基图需要**节点** + **链接（边）**两类数据：

**节点**（`nodes`）：

```
[
  { name: 'A' },
  { name: 'B' },
  { name: 'C' }
]
```

**链接**（`links`）：

```
[
  { source: 'A', target: 'B', value: 100 },
  { source: 'B', target: 'C', value: 80 }
]
```

- `source` / `target`：起 / 终节点名
- `value`：流量数值（决定边宽度）

---

## 布局

桑基图通常按列布局——节点排成若干列（如阶段 1 → 阶段 2 → 阶段 3），同列节点垂直堆叠，相邻列之间用带状曲线连接。

paradigm-chart 的 SankeyTree 在此基础上支持**树形布局**变体——节点按树深排列，更适合层级流向。

### 垂直对齐规则（流式居中）

所有列的节点以**主中间节点（最大汇聚枢纽）为锚点**进行垂直居中对齐，而非统一顶对齐。

- 主中间节点（如"营业收入"）作为全局垂直锚点，占据画布中央区域。
- **源节点列（汇聚侧）**：整组节点的垂直中心与主中间节点的垂直中心对齐，节点间保持最小间距，自然向上下散开，形成"收束汇入"的视觉效果。
- **下游各列（分散侧）**：整组节点的垂直中心同样与其上游父节点的垂直中心对齐，节点间保持最小间距，自然向上下散开，形成"展开分散"的视觉效果。布局与源节点列对称——源节点向枢纽收束，下游节点从枢纽展开。

> 桑基图的核心美感在于**收束—展开**的对称流动：左侧多个源节点汇聚到中心枢纽，右侧从枢纽向多个下游节点扇形展开。如果下游节点紧贴在一起不散开，会失去这种对称感和流动感。

### 节点避让规则

**汇节点（Sink Node）不允许与流入边重叠**。当汇节点的垂直位置导致节点矩形与其他边（非流入自身的边）发生视觉重叠时，必须将该节点**向下偏移**，直至与边带完全分离。

> 典型场景：中间节点（如毛利润、营业利润）的流出边向右侧多个汇节点分散，较小的汇节点可能恰好落在较大边带的路径上。此时小汇节点必须下移避让，保证每个节点的边界清晰可辨。

### 汇节点排序规则

同列汇节点按**正值在上、负值（支出）在下**的顺序垂直排列，不允许正值节点穿插在负值节点之间。

| 位置 | 节点类型 | 示例 |
| --- | --- | --- |
| **上方** | 正值汇节点 | 净利润 |
| **下方** | 负值 / 支出汇节点 | 营业成本、研发费用、管理费用、销售费用、财务费用、所得税 |

> 正负分区可以让读者一眼区分"钱去了哪里"和"留下了什么"，避免正值节点被夹在多个支出节点中间导致视觉混乱。

## 节点分类

| 类型 | 定义 | 典型节点 |
| --- | --- | --- |
| **源节点（Source Node）** | 最左列，只有流出、没有流入 | 消费电子、汽车电子等收入来源 |
| **中间节点（Intermediate Node）** | 中间列，既有流入又有流出，起汇聚再分配作用 | 营业收入、毛利润、营业利润 |
| **汇节点（Sink Node）** | 最右列，只有流入、没有流出 | 营业成本、研发费用、净利润、所得税 |

## 节点样式

| 元素 | 默认 |
| --- | --- |
| 形状 | 矩形 |
| 宽度（主中间节点） | **24px**（其他节点的 2 倍） |
| 宽度（其他节点） | **12px**（主中间节点的 1/2） |
| 主中间节点高度 | **240px**（作为全局尺度基准） |
| 节点最小间距 | 主中间节点高度 × **10%**（动态计算） |

> **节点宽度比 2:1**：仅**第一个中间节点**（即最大的汇聚枢纽，如"营业收入"）使用 24px 宽度，其余中间节点与源节点、汇节点一样使用 12px。主中间节点承担全局汇聚再分配职能，视觉上加宽以突出其枢纽地位；后续中间节点（如毛利润、营业利润）流量已分散，不再需要加宽强调。
>
> **节点间距**：同列节点之间的最小垂直间距 = 主中间节点高度 × 10%。例如主中间节点高度 240px 时，最小间距为 24px。该间距随主中间节点高度动态缩放，确保不同画布尺寸下节点间的呼吸感比例一致。

### 节点颜色（base 主题：固定 3 色；业务线主题大幅覆盖）

> ⚠️ **节点颜色被业务线主题大幅覆盖**——切了主题画桑基图时**必须**回主题文件确认实际配色：
>
> - **base 主题**：源/中间 = primary `#3366FF`，汇正 = price-up `#FF2436`（涨色，红），汇负 = price-down `#07AB4B`（跌色，绿）
> - **THS 主题** ⚠️ **语义反转**：源/中间 = `#FF2436`（涨色！），汇正 = `#FF9500`（橙），汇负 = `#3366FF`（蓝）
> - **Ainvest 主题**（海外配色）：源/中间 = `#265FFC`，汇正 = `#00B53C`（绿正），汇负 = `#FF381A`（红负）
>
> 详见 [base.md § 涨跌色](../themes/base.md#涨跌色delta-覆盖) / [ths.md](../themes/ths.md) / [ainvest.md § 1.7](../themes/ainvest.md#17-涨跌色--桑基图节点配色)。

节点颜色**不按系列色板取色**，固定使用 3 种颜色（下表是 **base 主题**值）：

| 节点类型 | base 主题颜色规则 | 示例 |
| --- | --- | --- |
| **源节点** | `color-visualization-primary`（`#3366FF`） | 收入来源（消费电子、汽车电子…） |
| **中间节点** | `color-visualization-primary`（与源节点同色） | 营业收入、毛利润、营业利润 |
| **汇节点（正值）** | `color-price-up`（`#FF2436`） | 净利润等正向结果 |
| **汇节点（负值 / 支出）** | `color-price-down`（`#07AB4B`） | 营业成本、费用、所得税等 |

> 源节点与中间节点使用同一颜色，强调"资金主流脉络"的连续性；汇节点根据数据正负区分——正值用 `color-price-up`，负值用 `color-price-down`，使流向终点的性质一目了然。
>
> **业务线主题可能反转此语义**（见上方 ⚠️ 警告）——切主题时务必查对应主题文件。

## 实现要点（库无关）

- **边宽按流量映射**：边宽度必须按 `value` 大小映射，否则失去"流量可视化"的核心价值。
- **节点列对齐**：同一阶段的节点垂直对齐成列，相邻列等距排布。
- **节点顺序优化**：用算法（如重心法）最小化边交叉，提升可读性——不要随意排列节点顺序。
- **边色取终节点色 20% 不透明度**：边颜色固定为终节点颜色 + `opacity: 0.2`，不允许渐变；交叠区域靠低透明度自然区分。
- **配色用类别**：节点按类别（业务线、阶段）上色；边色由终节点决定，无需单独配置。

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 边宽必须按流量映射，不要等宽 |
| ✅ | 节点顺序经过优化（最小化边交叉） |
| ✅ | 边色取终节点颜色 + 不透明度 20%，不允许渐变 |
| ❌ | 不要让所有边等宽——失去桑基图意义 |
| ❌ | 不要在节点过多（> 30）时还硬挤——考虑分页或聚合 |
| ❌ | 不要用桑基图展示无流向关系的数据（那是其他图的场景） |
