## Overview

瀑布图用于**展示数值的累积变化过程**——从期初值经过若干增减，得到期末值。每根柱代表一个阶段的增减或汇总，柱间用连接线串联，呈现累积逻辑。

典型应用：

- **财务报表**（期初余额 → 各项收支 → 期末余额）
- 价格构成（原料 + 人工 + 利润 = 售价）
- 数据归因（基础值 + 各影响因子 = 最终值）

---

## 数据结构

每个数据点：

```
{ name, value, type? }
```

- `name`：阶段名
- `value`：数值（正为增、负为减、汇总柱给绝对值）
- `type`：柱类型（可选）：
  - `positive`：正向增长
  - `negative`：负向减少
  - `total`：汇总柱（从 0 起绘制，代表当前累积值）

---

## 柱类型与颜色

| 类型 | 用途 | 颜色（待 PDF 确认） |
| --- | --- | --- |
| `total` | 期初 / 期末 / 阶段汇总 | 中性色或主色 |
| `positive` | 增长 | 涨色（`color-price-up`）或专用增长色 |
| `negative` | 减少 | 跌色（`color-price-down`）或专用减少色 |

> ⚠️ 增减是否用 A 股涨跌色（红涨绿跌），还是用非涨跌的"增色/减色"，**待设计师 PDF 确认**。代码示例用了非涨跌配色。

## 连接线

相邻两柱顶端通过连接线串联，清晰呈现累积关系。

| 属性 | 默认 |
| --- | --- |
| 显示 | 是 |
| 颜色 | 浅灰（可配置） |
| 粗细 | 1px |
| 线型 | 实线 |

## 总计柱

可在起点或终点单独标注"汇总"柱（位置 `start` / `end`），样式与中间增减柱区分（通常更深或独立色）。

## 柱体规范

宽高规则**部分覆盖**基础柱状图——**最大柱宽不同**，其他继承：

| 项 | 基础柱状图（[bar.md](bar.md)） | 瀑布图覆盖 |
| --- | --- | --- |
| **柱体最大宽度** | 32px（`size-bar-max`） | **64px** ⚠️ 瀑布图专属上限 |
| 单柱容器最大宽度（含柱距） | 48px | 按 2:1 比例推算 = **96px** |
| 柱距比（柱宽 : 间距） | 2:1 | 同（继承） |
| 高度占画布 | 95%（顶部留 5% 喘息） | 同（继承） |
| 柱顶圆角 | 0px（无圆角） | 同（继承，所有类型柱均直角） |

> **为什么瀑布图加宽到 64px**：瀑布图的柱数通常少（财务报表 5-10 根、归因 < 10 根），且每根承载叙事重量（期初 / 收 / 支 / 期末 / 累计）。加宽到 64px 让单柱视觉权重足够、读者聚焦每根的增减信息——比基础柱状图（多类别比较，柱数密、单柱权重轻）更需要单柱主导画面。
>
> **ECharts 实现**：`series.barMaxWidth: 64`（不是 32）。数据量增多时按 2:1 比例等比缩放。

所有类型柱（`positive` / `negative` / `total`）均**无圆角**，与其他柱状图一致。

## Y 轴 / 网格线 — ⚠️ 覆盖基础规则

瀑布图**覆盖** [base.md § 网格线](../themes/base.md#网格线grid) 和 [base.md § 坐标轴](../themes/base.md#坐标轴axes) 的相关默认规则：

| 项 | base 主题（默认） | 瀑布图覆盖 |
| --- | --- | --- |
| Y 方向非 0 轴的水平网格线 | 默认 5 条 | **不显示** |
| **Y 轴标签（刻度值）** | 默认显示 | **不显示**（与网格线同隐） |
| 0 轴线 | 加深 `#858585` | **保留并加深显示**（`color-visualization-divider-deep`） |
| X 方向网格线 | 默认隐藏 | 同（继承） |
| **柱顶数据标签** | 继承基础柱状图（默认显示） | **必显**——替代 Y 轴标签承担数值标注职能 |

> **为什么 Y 轴标签必须与网格线同隐**：Y 轴标签是"刻度值"，它**依赖网格线作为视觉锚定**——读者沿着某条网格线水平延伸，才能把标签数值对应到柱体高度上。一旦关掉网格线，标签就成了**悬浮在轴侧的孤立数字**，无法和任何柱体建立视觉关联。
>
> 瀑布图的数值参照由**柱顶数据标签**直接承担（每根柱显式标出自己的增减值），不再需要 Y 轴标签的间接刻度参照——这是"局部精确 vs 全局粗略"的取舍：瀑布图的核心叙事是阶段间变化（局部），不是整体水位线（全局）。
>
> **为什么覆盖网格线本身**：视觉重心在"柱与柱的**连接线** + **0 轴**作为正负参考"。多条水平网格线会与连接线视觉冲突，弱化累积叙事。只留 0 轴线作为增/减柱的中性参照，画面更聚焦在阶段间变化。
>
> **ECharts 实现**：
>
> ```js
> yAxis: {
>   splitLine: { show: false },     // 关常规网格线
>   axisLabel: { show: false },     // 关 Y 刻度标签 ← 与网格线必须同设
>   axisLine: { show: false },      // 顺手关 Y 轴线（可选，但 Y 标签都没了轴线也无意义）
>   axisTick: { show: false }
> }
>
> // 在任一 series 上挂 markLine 单独画 0 轴
> series: [{
>   markLine: {
>     silent: true, symbol: 'none',
>     data: [{ yAxis: 0 }],
>     lineStyle: { color: '#858585', width: 1, type: 'solid' }
>   },
>   // 柱顶数据标签必显
>   label: { show: true, position: 'top', formatter: '{c}' }
> }]
> ```
>
> 详见 [echarts-implementation-hints.md 陷阱 9](../echarts-implementation-hints.md)。

## 实现要点（库无关）

- **三类柱必须视觉区分**：`positive` / `negative` / `total` 三类柱用不同颜色，确保增减汇总一眼分辨。
- **连接线不是装饰，是叙事线**：连接线把"前一柱顶 → 后一柱底"的逻辑链显式画出，缺了它瀑布图就退化为普通柱图。
- **汇总柱单独样式**：汇总柱（期初 / 期末 / 中间汇总）颜色应与增减柱有别，通常更深或用主色。
- **数据顺序 = 叙事顺序**：数据顺序决定累积路径，不能随意打乱。

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 三类柱（增 / 减 / 汇总）颜色明显区分 |
| ✅ | 连接线显示，串联累积关系 |
| ✅ | 数据按时间 / 叙事顺序排列 |
| ✅ | 汇总柱（期初 / 期末）单独样式 |
| ✅ | **柱体无圆角**——四角直角 |
| ❌ | 不要把所有柱用同色——失去增减叙事 |
| ❌ | 不要隐藏连接线——丢掉累积关系信息 |
| ❌ | 不要给瀑布柱加圆角——柱体应为直角 |
| ❌ | 不要用瀑布图展示非累积数据（那是分组柱状图的场景） |
| ❌ | **不要画 Y 方向常规网格线**——只保留 0 轴线作为正负参考 |
| ❌ | **不要保留 Y 轴刻度标签**——网格线隐藏后标签悬空无锚定参照；数值由**柱顶 label** 直接显式标出 |
