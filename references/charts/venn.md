# 韦恩图（Venn）

## Overview

韦恩图（Venn Diagram）用于展示**集合之间的交集关系**——每个圆代表一个集合，圆与圆之间的重叠区域代表交集。

典型应用：

- 用户特征分析（用户 A ∩ 用户 B = 共同特征用户）
- 市场细分重叠（高净值 ∩ 年轻 ∩ 一线城市）
- 标签 / 兴趣交集

---

## 数据结构

每个数据点：

```
{ name, value?, sets? }
```

- `name`：集合或交集名（如 `A` / `A&B` / `A&B&C`）
- `value`：该区域的数值（影响视觉占比）
- `sets`：该交集涉及的集合数组（如 `['A', 'B', 'C']`），用于标识 3+ 维度的复合交集

---

## 维度数

| 维度 | 说明 |
| --- | --- |
| **2** | 两个集合的简单交集（常规） |
| **3** | 三个集合（经典韦恩图，3 圆） |
| **4** | 四个集合（椭圆形或非圆形） |
| **5-7** | 高维韦恩图（几何上需特殊形态；**视觉复杂度高，谨慎使用**） |

---

## 配色

- 每个集合一个颜色，从 [tokens.md — 可视化色板](../tokens.md) 取色
- 交集区域颜色通常是涉及集合的颜色叠加（透明度混合）
- 可通过自定义颜色映射给每个集合 / 交集独立配色

---

## 标签

- 默认显示在每个区域中心
- 内容：集合名 + 数值（可配置）
- 字号 / 颜色：见 [数据标签规范](../components/data-label.md)

## 标题（可选）

可在韦恩图上方显示主标题 + 副标题，用于场景说明。

---

## 实现要点（库无关）

- **维度数 ≤ 4**：超过 4 个集合的韦恩图几何上需要椭圆 / 非圆形，视觉复杂；考虑改用其他图（如 UpSet 图）。
- **集合大小用 `value` 映射**：让每个区域的视觉占比对齐数据占比，而不是纯结构示意图。
- **交集要明确**：交集区域必须用 `sets` 字段声明涉及哪些集合，避免视觉模糊。
- **颜色透明度低**：每个集合用低透明度（20-40%），让交集区域（叠加效果）能清晰呈现。

---

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 集合数控制在 2-4 维 |
| ✅ | 用 `value` 表达每个区域的占比，让视觉权重对齐数据 |
| ✅ | 各集合颜色低透明度（20-40%），便于交集叠加显示 |
| ❌ | 不要超过 4 个集合——视觉会很乱 |
| ❌ | 不要用满色（100% 透明度）——交集区域看不出叠加 |
| ❌ | 不要用韦恩图展示非"集合关系"的数据 |
