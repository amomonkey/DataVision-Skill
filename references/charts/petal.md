# 花瓣图（Petal）

## Overview

花瓣图（Petal）是一种放射状的形态图——多个"花瓣"从中心向外辐射，每个花瓣代表一个维度，花瓣的长度 / 宽度可映射数值。

典型应用：

- 多维评分（每片花瓣 = 一个维度）
- 周期性数据（按月份 / 时辰展开为花瓣）
- 装饰性可视化（财报亮点、能力雷达的变体）

> 与雷达图的区别：雷达图是一个连续多边形；花瓣图是**独立的、可单独配置的花瓣**，每片花瓣形态可不同。

---

## ⚠️ 资料受限说明

代码层面 petal 资料较少：

- `src/extension/series/petal/action.ts`：仅注册 `dvPetalShowHidden` 显示/隐藏 action
- `src/theme/pc/series/petal.ts`：主题解析（识别 `series.type === 'custom'` 且 `dvType === 'petal'`）
- 没有独立的 series model / view，花瓣图是基于 ECharts `custom` series 实现

**这意味着**：花瓣图的具体形态、样式、数据结构由业务方在 `custom` series 的 `renderItem` 函数中自定义，没有标准化的规范字段。本文档仅能给出概念性描述。

---

## 数据结构（概念）

由于基于 `custom` series，数据结构由业务定义。常见形态：

```
[
  { name: '维度A', value: 80, angle: 0 },
  { name: '维度B', value: 60, angle: 60 },
  { name: '维度C', value: 90, angle: 120 },
  ...
]
```

- `name`：花瓣名
- `value`：数值（映射花瓣长度或宽度）
- `angle`：花瓣角度（决定方位）

---

## 关键设计变量

| 变量 | 说明 |
| --- | --- |
| 花瓣数 | 通常 4-12 片（少了像扇形，多了像雷达） |
| 花瓣形态 | 椭圆 / 水滴 / 自定义 path |
| 花瓣长度映射 | 按 value 映射（最大值 = 完全展开） |
| 花瓣宽度 | 固定或按次维度映射 |
| 中心元素 | 可在中心放 logo / 文字 / 数字 |
| 颜色 | 每片花瓣一色（顺序色板） / 渐变 |

## 交互

| 行为 | 默认 |
| --- | --- |
| 显示 / 隐藏 | `dvPetalShowHidden` action 可控制单片花瓣的显示 / 隐藏 |
| 悬停 | Tooltip 显示花瓣对应维度的数值 |

## 实现要点（库无关）

- **花瓣数适中**：4-8 片视觉最佳；超过 12 片会变得像雷达图或太阳花，失去"花瓣"特征。
- **花瓣旋转对齐**：所有花瓣以中心为原点旋转排列，间距均匀；不要让花瓣大小不一时强行旋转，会破坏对称感。
- **中心可放关键信息**：中心区域常用于显示总分 / logo / 关键指标，强化"焦点"感。
- **基于 custom series 实现**：因为是 `custom` series，所有样式由业务在 `renderItem` 里自绘——这意味着每个项目可能形态略不同；用 paradigm-chart 主题（`dvType: 'petal'`）来统一风格。

---

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 花瓣数 4-8 片，对称排列 |
| ✅ | 中心区域放关键信息（总分 / logo） |
| ✅ | 每片花瓣按维度上色 |
| ❌ | 不要让花瓣数过多（> 12）——失去花瓣形态 |
| ❌ | 不要让花瓣大小悬殊到破坏对称——选合适的 value 映射 |
| ❌ | 不要把花瓣图当雷达图用——前者是离散花瓣，后者是连续多边形 |
