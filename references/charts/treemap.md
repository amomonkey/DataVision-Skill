# 矩形树图（Treemap）

## Overview

矩形树图用于展示**层级结构**与**数值占比**——每个节点是一个矩形，矩形面积代表数值大小，可下钻到子层级。

典型应用：

- 文件 / 目录占用可视化
- 行业 / 板块市值占比
- 产品 / 品类销售构成
- 任何"层级 + 占比"的场景

---

## 数据结构

树形递归：

```
{ name, value, children: [...] }
```

- `name`：节点名
- `value`：数值（决定矩形面积）
- `children`：子节点数组（可选）

支持自定义节点样式 `itemStyle` / `label` / `emphasis`。

## 布局方向

| 布局 | 说明 |
| --- | --- |
| `horizontal` | 水平方向优先切分 |
| `vertical` | 垂直方向优先切分 |

## 节点样式

| 元素 | 默认 |
| --- | --- |
| 颜色 | 系列色 / 函数计算 |
| 边框颜色 | 浅色（视觉分隔） |
| 边框宽度 | 配置 |
| 节点间距（`gapWidth`） | 配置（影响视觉密度） |

---

## 标签

矩形树图的标签布局有 3 种特殊类型（paradigm-chart 增强）：

| 类型 | 说明 |
| --- | --- |
| `twoLineBasic` | 双行布局（基础）：名称 + 数值各占一行 |
| `twoLineCenter` | 双行居中布局 |
| `staticCenter` | 静态居中布局（单行） |

**富文本配置**（节点足够大时显示更丰富的标签）：

| 配置 | 说明 |
| --- | --- |
| `dvMinFontSize` | 最小字号（节点小到一定程度时不再显示） |
| `dvLineHeight` | 行高 |
| `dvValueFontDeviation` | 数值字号偏差（数值比名称略大 / 小） |

---

## 上层节点标签（`upperLabel`）

下钻后，上层节点（祖先节点）的标签可单独显示在矩形顶部：

| 属性 | 默认 |
| --- | --- |
| 显示 | 可配置 |
| 高度 | 配置（如 24px） |
| 位置 | 顶部 |
| 截断 | 支持省略号 |

---

## 下钻（Breadcrumb）

点击节点进入下一层，可显示面包屑导航：

| 属性 | 默认 |
| --- | --- |
| 显示 | 可配置 |
| 项样式 | 颜色、边框、阴影、文字 |
| 空白项宽度 | `emptyItemWidth` 配置 |

---

## 增强配置（paradigm-chart 专属）

| 配置 | 说明 |
| --- | --- |
| `dvMinPercent` | 最小显示比例（小于该比例的节点按此比例显示，避免极小节点消失） |
| `dvLabelType` | 标签布局类型（见上方） |
| `dvLabelShowTimer` | 下钻后标签显示动画时间（ms） |

---

## 动画

- 入场动画时长 / 缓动可配置
- 下钻 / 上钻时矩形变换动画

---

## 实现要点（库无关）

- **极小节点兜底**：用 `dvMinPercent` 给极小数值的节点一个最小显示比例，避免完全消失。
- **标签自适应**：节点足够大时显示双行（名称 + 数值），小节点只显示名称或不显示；最小字号兜底。
- **下钻路径要可回溯**：面包屑导航要清晰，让用户知道当前在哪一层、可以返回。
- **颜色按层级或类别**：同层兄弟节点用同色板的不同色，或者按深度递减明度。
- **gap 不要太大**：节点间距大会浪费空间、削弱占比直观性；通常 1-2px。

---

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 极小节点用 `dvMinPercent` 兜底，避免消失 |
| ✅ | 标签按节点大小自适应（大节点双行、小节点单行 / 不显示） |
| ✅ | 显示面包屑，确保下钻路径可回溯 |
| ✅ | 同层节点颜色按色板分配 |
| ❌ | 不要让节点间距过大（> 4px）——浪费空间 |
| ❌ | 不要在所有节点上强制显示数值（小节点装不下） |
| ❌ | 不要用矩形树图展示无层级、纯类别的占比（那是饼图 / 环图的场景） |
