# 标记系列（Marker / dvMarker）

>
> 注：`dvMarker` 是**辅助标记系列**，不是独立的图表类型——它叠加在其他图表（柱 / 折线等）上，提供额外的标记、旗标、曲线指针等元素。

## Overview

标记系列用于在主图表上叠加**额外的标识 / 注释 / 指针**：

- **旗标（Flag）**：在数据点旁挂一面"旗子"，标注重要事件 / 节点（如重大新闻日）
- **曲线指针（Curved Pointer）**：用曲线箭头从一处指向另一处，强调关联或流向
- **矩形 / 标签**：自由矩形标注块，支持垂直位置自适应

典型应用：

- K 线图叠加"利好 / 利空"事件标签
- 折线图标注"政策发布日"等关键时间点
- 多图联动时用曲线箭头连接相关数据点

---

## 主要能力

### 旗标（FlagLayoutHelper）

| 能力 | 说明 |
| --- | --- |
| 位置 | 锚定到数据点（X / Y 坐标） |
| 旗杆 | 从锚点向上 / 向下的细线 |
| 旗面 | 旗杆顶端的矩形 / 标签，内放文字或图标 |
| 布局 | 多个旗标时自动避让，防重叠 |

### 曲线指针（curvedPointerLayoutHelper）

| 能力 | 说明 |
| --- | --- |
| 起点 / 终点 | 两个数据点（或自定义坐标） |
| 曲线形态 | 平滑贝塞尔曲线，端点带箭头 |
| 曲率 | 可调，避免穿越其他图形 |

> 实现细节见 `docs/design-specification/curvedPointer-implementation-guide.md`（paradigm-chart 内部）。

### 矩形垂直位置调整（adjustRectVerticalPosition）

| 能力 | 说明 |
| --- | --- |
| 自适应 | 标签 / 矩形垂直位置按可用空间自动调整 |
| 防溢出 | 避免标签超出图表绘制区上下边界 |

---

## 数据结构（概念）

标记系列通常作为额外 series 叠加到图表，数据结构按标记类型而异：

**旗标**：

```
[
  { coord: [x, y], text: '事件A', direction: 'up' },
  { coord: [x, y], text: '事件B' },
  ...
]
```

**曲线指针**：

```
[
  { from: [x1, y1], to: [x2, y2], label: '关联' },
  ...
]
```

## 样式

| 元素 | 默认 |
| --- | --- |
| 旗杆颜色 | 中性灰或主题色 |
| 旗杆粗细 | 1px |
| 旗面背景 | 浅色（高对比） |
| 旗面文字字号 | 10-12px（小） |
| 曲线指针颜色 | 主色或中性色 |
| 曲线指针箭头 | 三角形，端点 |

---

## 实现要点（库无关）

- **辅助叠加，不抢主图**：标记是辅助元素，颜色 / 字号要克制，不要让标记色压过主图数据色。
- **多标记自动避让**：多个旗标在同一区域时必须自动错位 / 调整位置，避免互相覆盖。
- **垂直位置自适应**：标签靠近图表上下边界时自动反向（向上不下）或缩进，防止溢出。
- **曲线指针避免穿越**：曲率设计要避免曲线穿过主图的关键数据（柱 / 折线节点）。
- **锚点必须明确**：每个标记必须有清晰的"锚点"（指向哪个数据点 / 坐标），不能浮在空白处。

---

## Do & Don't

| | 规则 |
| --- | --- |
| ✅ | 标记颜色 / 字号克制，不抢主图 |
| ✅ | 多标记自动避让，不互相覆盖 |
| ✅ | 标签靠近边界时自动调整位置 |
| ✅ | 曲线指针曲率避免穿越关键数据 |
| ❌ | 不要让标记 / 旗标颜色比主图数据更鲜艳 |
| ❌ | 不要在密集数据区放过多标记——视觉过载 |
| ❌ | 不要让多个标记互相覆盖——失去标记意义 |
